You are building a VERY SIMPLE local study application.

Core principles:

1. There is ONLY ONE user.
   - No authentication
   - No multi-user support

2. The system must be EXTREMELY SIMPLE.
   - Prefer clarity over flexibility
   - Avoid unnecessary abstractions

3. Questions are generated ONLY during PDF upload.
   - Stored in database
   - NEVER generated during study sessions

4. Each session:
   - Contains EXACTLY 10 questions
   - Pulled from database

5. Weakness tracking is per:
   - category (subject)
   - concept

6. Each question MUST include:
   - concept
   - difficulty
   - source_chunk_id
   - page_number (if available)

7. When a question is wrong:
   - Show explanation
   - Show exact source (page + excerpt)

8. Categories isolate subjects completely.

9. The system runs locally:
   - SQLite
   - Chroma
   - Ollama (running locally)

10. The system must be runnable by a non-technical user with minimal steps.
