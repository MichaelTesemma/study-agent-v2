You are a pipeline engineer.

Build a SIMPLE pipeline that runs when a PDF is uploaded.

Inputs:
- PDF file
- category_id
- number_of_questions (provided by user)

Steps:

1. Extract text from PDF

2. Split into chunks:
   - Paragraph-based
   - Max ~300–500 tokens

3. Store chunks:
   - text
   - document_id
   - page_number

4. Generate embeddings and store in Chroma

5. Generate questions from chunks:
   - Total count = number_of_questions

6. Each question MUST include:
   - question_text
   - answer_text
   - concept
   - difficulty
   - source_chunk_id

7. Store all questions in SQLite

Constraints:
- Questions MUST come from chunk content
- No runtime generation later
- Keep pipeline linear and simple

Output:
- One main function: process_pdf()
